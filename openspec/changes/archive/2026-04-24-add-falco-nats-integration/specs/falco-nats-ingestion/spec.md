# Spec: falco-nats-ingestion

## Overview

Falco runtime security events are ingested into ServiceRadar's NATS JetStream pipeline
via Falcosidekick's NATS output using JWT `.creds` file authentication. Each Falcosidekick
instance is onboarded via the existing CollectorPackage pipeline.

## ADDED Requirements

### Requirement: Direct NATS Ingestion via Falcosidekick

Falcosidekick SHALL connect directly to ServiceRadar's NATS JetStream and publish Falco
events using JWT `.creds` file authentication.

The integration SHALL:
1. Use Falcosidekick's NATS output with `.creds` file support (upstream patch)
2. Publish raw Falco event JSON to the `events.falco.raw` NATS subject
3. Verify the NATS server certificate against a provided CA cert

#### Scenario: Falcosidekick publishes a Falco event to NATS

- **GIVEN** Falcosidekick is configured with a valid `.creds` file and CA cert
- **AND** NATS JetStream is running with the corresponding account JWT
- **WHEN** Falco detects a runtime security event
- **THEN** Falcosidekick publishes the event JSON to `events.falco.raw`
- **AND** the event is stored in the JetStream `events` stream

#### Scenario: Falcosidekick reconnects after NATS disruption

- **GIVEN** Falcosidekick is running and connected to NATS
- **WHEN** NATS becomes temporarily unreachable
- **THEN** the NATS client automatically reconnects when connectivity is restored
- **AND** subsequent events are published normally

#### Scenario: Falcosidekick rejects invalid credentials

- **GIVEN** Falcosidekick is configured with an expired or revoked `.creds` file
- **WHEN** Falcosidekick attempts to connect to NATS
- **THEN** the connection is rejected by the NATS server
- **AND** Falcosidekick logs the authentication error

### Requirement: Falcosidekick Collector Type in Onboarding

The onboarding system SHALL support a `falcosidekick` collector type that provisions
NATS JWT credentials with publish permissions scoped to Falco event subjects.

#### Scenario: Provisioning a falcosidekick collector package

- **GIVEN** an admin creates a CollectorPackage with collector_type `:falcosidekick`
- **WHEN** the ProvisionCollectorWorker processes the package
- **THEN** NATS JWT credentials are generated with publish permission for `events.falco.>`
- **AND** subscribe permission for `_INBOX.>`
- **AND** the credentials are encrypted and stored in the package

#### Scenario: Downloading a falcosidekick bundle

- **GIVEN** a provisioned falcosidekick CollectorPackage in `:ready` state
- **WHEN** the bundle is downloaded via enrollment token
- **THEN** the tarball contains `creds/nats.creds` and `certs/ca-chain.pem`

### Requirement: TLS Transport Security

Falcosidekick SHALL connect to NATS using TLS with CA certificate verification.

#### Scenario: Falcosidekick verifies NATS server certificate

- **GIVEN** a CA certificate file configured via `nats.cacertfile`
- **WHEN** Falcosidekick connects to NATS over TLS
- **THEN** the server certificate is verified against the CA
- **AND** the connection is established only if verification succeeds

### Requirement: Per-Cluster Credential Isolation

Each Falcosidekick instance SHALL use unique NATS credentials generated by the
onboarding pipeline, enabling independent revocation per cluster.

#### Scenario: Revoking credentials for a single cluster

- **GIVEN** Falcosidekick instances on Cluster A and Cluster B each have unique credentials
- **WHEN** the admin revokes the credentials for Cluster A
- **THEN** Cluster A's Falcosidekick can no longer publish to NATS
- **AND** Cluster B's Falcosidekick continues operating normally

### Requirement: User Documentation

The system SHALL provide documentation for integrating Falco with ServiceRadar.

#### Scenario: New user follows setup guide

- **GIVEN** a user with Falco and Falcosidekick deployed on a Kubernetes cluster
- **AND** ServiceRadar with NATS JetStream running
- **WHEN** the user follows the `docs/docs/falco-integration.md` guide
- **THEN** they can create a falcosidekick collector package
- **AND** configure Falcosidekick Helm values with the NATS credentials
- **AND** verify Falco events appear on the `events.falco.raw` NATS subject
