## MODIFIED Requirements
### Requirement: Agent Effective Configuration Fetch
The `serviceradar-agent` MUST fetch its effective runtime configuration from the control plane via gRPC when no local override exists. The agent SHALL request configuration from agent-gateway after a successful `Hello`. The control plane SHALL return compiled configuration generated by web-ng/core-elx, including embedded runtimes (sync integrations, discovery/mapping, SNMP polling), plugin assignments, and command bus settings.

#### Scenario: Fetch config on startup
- **GIVEN** a registered agent without a local override config
- **WHEN** the agent process starts
- **THEN** it requests its effective configuration from agent-gateway
- **AND** receives an effective configuration payload compiled by the control plane
- **AND** the response includes a configuration version identifier

#### Scenario: Config fetch failure with fallback
- **GIVEN** an agent starting up
- **AND** the control plane is unreachable
- **WHEN** the agent attempts to fetch configuration
- **THEN** it retries with exponential backoff (max 5 attempts)
- **AND** falls back to cached configuration if available
- **AND** uses a safe default config if no cache exists

#### Scenario: Config fetch timeout
- **GIVEN** an agent attempting to fetch configuration
- **WHEN** the request takes longer than 30 seconds
- **THEN** the request times out
- **AND** the agent proceeds with fallback logic

#### Scenario: Local override takes precedence
- **GIVEN** an agent has a local override config present
- **WHEN** the agent starts
- **THEN** it uses the local override
- **AND** it does not block startup on a control plane config fetch
